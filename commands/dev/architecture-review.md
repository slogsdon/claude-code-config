# Architecture Review

Conduct comprehensive architecture review adapted to the current project context.

## Review Process
1. **Project Context Analysis**
   - Read existing CLAUDE.md for project-specific patterns
   - Analyze package.json/pyproject.toml for tech stack
   - Review project structure and existing conventions

2. **Architecture Assessment**
   - **Scalability**: Can the architecture handle growth?
   - **Maintainability**: Is the code organized for long-term maintenance?
   - **Performance**: Are there obvious bottlenecks?
   - **Security**: Are security best practices followed?
   - **Testability**: Is the architecture test-friendly?

3. **Technology Stack Review**
   - Framework choices and versions
   - Dependency management
   - Infrastructure considerations
   - Integration patterns

4. **Recommendations**
   - Immediate improvements (low effort, high impact)
   - Medium-term refactoring opportunities
   - Long-term architectural evolution
   - Risk mitigation strategies

**Adapt to project:** Always check project-specific conventions first, then provide recommendations that fit the existing stack and team preferences.
