# Generate Documentation

Create comprehensive documentation adapted to project type and existing documentation patterns.

## Documentation Strategy
1. **Project Context**
   - Review existing documentation structure
   - Identify documentation gaps
   - Match existing style and format

2. **Documentation Types**
   - **README**: Project overview, setup, basic usage
   - **API Documentation**: Endpoints, parameters, examples
   - **Architecture Documentation**: System design, data flow
   - **User Documentation**: Feature guides, tutorials
   - **Developer Documentation**: Contributing guide, code conventions

3. **Content Generation**
   - Auto-generate from code comments and annotations
   - Create examples and use cases
   - Include troubleshooting guides
   - Add diagrams and visual aids

4. **Maintenance Strategy**
   - Documentation-as-code approach
   - Automated updates from CI/CD
   - Review and update schedule
   - Version control integration

**Output:** Comprehensive documentation following project conventions
