# Skill: /level-up [domain]

- Step 1: Read vault evidence of skill level — projects built, concepts documented, tools used, writing quality, complexity of problems tackled
- Step 2: Assess where skill level currently sits across key domains (AI/LLM tooling, product thinking, developer advocacy, writing, Rust, etc.)
- Step 3: Identify the next level in each domain — what would mastery look like one step up?
- Step 4: Recommend 3 specific growth actions — which vault commands would help, what to build or write, what to read
- Output: Assessment by domain + growth plan with specific actionable next steps
