# Scope Requirements

Analyze and scope requirements for a new feature or product using structured methodology.

## Process
1. **Stakeholder Analysis**
   - Identify primary and secondary stakeholders
   - Map user personas and use cases
   - Define success criteria

2. **Requirements Gathering**
   - Functional requirements (what the system must do)
   - Non-functional requirements (performance, security, usability)
   - Business requirements (ROI, timeline, constraints)

3. **Scope Definition**
   - MVP vs full feature set
   - Phase 1, 2, 3 breakdown
   - Dependencies and blockers
   - Resource requirements

4. **Documentation Template**
   - User stories with acceptance criteria
   - Technical requirements
   - Risk assessment
   - Timeline estimates

**Input needed:** Brief description of the feature/product idea
**Output:** Comprehensive requirements document with scope recommendations
